# Glossário

Este glossário define os termos técnicos e conceitos específicos utilizados neste documento de requisitos. Os termos estão organizados em ordem alfabética para facilitar a consulta.

## A

**ACID (Atomicity, Consistency, Isolation, Durability)**: Conjunto de propriedades que garantem confiabilidade em transações de banco de dados.

**AES-256**: Algoritmo de criptografia simétrica considerado padrão de segurança para proteção de dados sensíveis.

**Android Studio**: Ambiente de desenvolvimento integrado (IDE) oficial para o desenvolvimento de aplicações Android, fornecido pelo Google.

**AOF (Append Only File)**: Método de persistência do Redis que registra todas as operações de escrita em um arquivo.

**API (Application Programming Interface)**: Conjunto de definições e protocolos para construir e integrar software de aplicação, permitindo comunicação entre diferentes componentes de software.

**APM (Application Performance Monitoring)**: Sistema de monitoramento que acompanha performance e saúde de aplicações em tempo real.

**Aplicativo Mobile**: Software desenvolvido especificamente para execução em dispositivos móveis como smartphones e tablets.

**Auto-scaling**: Capacidade de ajustar automaticamente recursos computacionais baseado na demanda de uso.

## B

**Backup**: Cópia de segurança de dados armazenada separadamente para recuperação em caso de perda ou corrupção.

**Balanceamento de Carga**: Distribuição de requisições entre múltiplos servidores para otimizar performance e disponibilidade.

**Banco de Dados**: Sistema organizado para coleta, armazenamento e recuperação de informações digitais.

**bcrypt**: Algoritmo de hash criptográfico especificamente projetado para senhas, incluindo salt automático.

**betSTOP**: Nome do sistema objeto deste documento - aplicativo mobile terapêutico para tratamento do vício em jogos de azar online.

**Bundle**: Pacote compilado de código e recursos de uma aplicação pronto para distribuição.

## C

**Cache**: Sistema de armazenamento temporário de dados frequentemente acessados para melhorar performance.

**CDN (Content Delivery Network)**: Rede distribuída de servidores que entrega conteúdo web de forma otimizada baseada na localização geográfica.

**CI/CD (Continuous Integration/Continuous Deployment)**: Prática de desenvolvimento de software que envolve integração contínua de código e implantação automatizada de aplicações.

**Circuit Breaker**: Padrão de design que previne falhas em cascata interrompendo chamadas para serviços com problemas.

**Cloud Computing**: Fornecimento de serviços de computação através da internet, incluindo armazenamento, processamento e software.

**Cloud SQL**: Serviço de banco de dados relacional totalmente gerenciado oferecido pelo Google Cloud Platform.

**Clustering**: Agrupamento de múltiplos servidores que trabalham juntos como um sistema único para alta disponibilidade.

**Container**: Tecnologia de virtualização que permite empacotar aplicações e suas dependências em um ambiente isolado e portátil.

**Containerização**: Processo de empacotamento de aplicações em containers para facilitar implantação e escalabilidade.

## D

**Dart**: Linguagem de programação desenvolvida pelo Google, utilizada pelo framework Flutter.

**Delta Sync**: Sincronização que transfere apenas as mudanças (diferenças) entre versões de dados.

**Docker**: Plataforma de containerização que permite criar, implantar e executar aplicações em containers.

**DPI (Dots Per Inch)**: Medida de densidade de pixels em uma tela, afetando a resolução e qualidade visual.

## F

**Failover**: Processo automático de transferência para um sistema backup quando o sistema principal falha.

**Feature Flag**: Técnica que permite ativar ou desativar funcionalidades remotamente sem atualizações de código.

**Flutter**: Framework de desenvolvimento de aplicações móveis criado pelo Google, que permite criar aplicativos nativos para iOS e Android usando uma única base de código.

## G

**Garbage Collector**: Processo automático de gerenciamento de memória que remove objetos não utilizados.

**GCP (Google Cloud Platform)**: Plataforma de computação em nuvem oferecida pelo Google, fornecendo serviços de infraestrutura, plataforma e software.

**Gin**: Framework web minimalista e rápido para construção de APIs REST em linguagem Go.

**Git**: Sistema de controle de versão distribuído para rastrear mudanças no código-fonte durante o desenvolvimento de software.

**GitHub**: Plataforma de hospedagem de código-fonte que utiliza o sistema de controle de versão Git.

**GitHub Actions**: Plataforma de automação do GitHub para CI/CD e workflows de desenvolvimento.

**GKE (Google Kubernetes Engine)**: Serviço gerenciado de Kubernetes oferecido pelo Google Cloud Platform.

**Go (Golang)**: Linguagem de programação desenvolvida pelo Google, conhecida por alta performance e concorrência.

**Goroutines**: Threads leves e eficientes da linguagem Go para programação concorrente.

**Gzip**: Algoritmo de compressão de dados usado para reduzir tamanho de arquivos e tráfego de rede.

## H

**Hash**: Função matemática que converte dados de entrada em uma string de tamanho fixo para segurança.

**Hot Reload**: Funcionalidade do Flutter que permite ver mudanças no código instantaneamente durante desenvolvimento.

**HTTPS**: Protocolo de comunicação seguro que adiciona criptografia TLS ao HTTP padrão.

## I

**IDE (Integrated Development Environment)**: Ambiente de desenvolvimento integrado que fornece ferramentas abrangentes para desenvolvimento de software.

**Infrastructure as Code**: Prática de gerenciar infraestrutura através de código versionado e automatizado.

**iOS**: Sistema operacional móvel desenvolvido pela Apple para seus dispositivos iPhone e iPad.

**ISO 27001**: Padrão internacional para sistemas de gestão de segurança da informação.

## J

**Jogos de Azar**: Atividades que envolvem apostas em eventos com resultados incertos, onde o resultado depende principalmente da sorte.

**JSON (JavaScript Object Notation)**: Formato leve de intercâmbio de dados baseado em texto, fácil de ler e escrever.

**JWT (JSON Web Token)**: Padrão para criação de tokens de acesso seguros contendo informações de autenticação.

## K

**Kubernetes**: Plataforma de orquestração de containers para automação de deploy, escala e gestão de aplicações.

## L

**Lazy Loading**: Técnica de carregamento sob demanda que adia o carregamento de recursos até serem necessários.

**LGPD (Lei Geral de Proteção de Dados)**: Lei brasileira que regula o tratamento de dados pessoais por organizações.

**Load Balancer**: Sistema que distribui requisições de rede entre múltiplos servidores para otimizar recursos.

## M

**Master-Slave**: Arquitetura de replicação onde um servidor principal (master) sincroniza dados com servidores secundários (slaves).

**Microserviços**: Arquitetura que estrutura aplicação como conjunto de serviços pequenos e independentes.

**Mobile First**: Abordagem de design que prioriza a experiência em dispositivos móveis antes de adaptar para outros dispositivos.

**MySQL**: Sistema de gerenciamento de banco de dados relacional de código aberto amplamente utilizado.

## N

**Notificação Push**: Mensagem enviada por um aplicativo para o dispositivo do usuário, mesmo quando o aplicativo não está ativo.

## O

**OAuth 2.0**: Protocolo de autorização que permite acesso seguro a recursos sem compartilhar credenciais.

**Observabilidade**: Capacidade de entender o estado interno de um sistema através de dados externos que ele produz.

## P

**Payload**: Dados transportados em uma transmissão de rede, excluindo cabeçalhos e metadados.

**Persona**: Representação fictícia de um usuário real, baseada em dados e características de usuários reais, utilizada no design de produtos.

**Point-in-time Recovery**: Capacidade de restaurar banco de dados para qualquer momento específico no passado.

**Pseudoaleatório**: Algoritmo que produz sequências que aparecem aleatórias mas são determinísticas.

**Psicologia Comportamental**: Área da psicologia que estuda como o comportamento é aprendido e modificado através de estímulos e reforços.

## R

**Rate Limiting**: Técnica de controle que limita número de requisições por usuário em período determinado.

**RBAC (Role-Based Access Control)**: Sistema de controle de acesso baseado em funções e permissões de usuários.

**Redis**: Sistema de armazenamento de estruturas de dados em memória, usado como banco de dados, cache e message broker.

**Reenquadramento Cognitivo**: Técnica terapêutica que envolve identificar e modificar padrões de pensamento negativos ou disfuncionais.

**Replicação**: Processo de manter cópias idênticas de dados em múltiplos locais para redundância.

**Requisito Funcional (RF)**: Especificação do que o sistema deve fazer, descrevendo as funções e serviços que o sistema deve fornecer.

**Requisito Não Funcional (RNF)**: Especificação de critérios que podem ser usados para julgar a operação de um sistema, como desempenho, segurança e usabilidade.

**REST (Representational State Transfer)**: Estilo arquitetural para desenvolvimento de APIs web baseado em HTTP.

**Rollback**: Processo de reverter sistema ou dados para estado anterior quando problemas ocorrem.

**RS256**: Algoritmo de assinatura digital usando RSA com SHA-256 para segurança de tokens.

## S

**Salt**: Dados aleatórios adicionados a uma senha antes de aplicar função hash para aumentar segurança.

**Sharding**: Técnica de particionamento horizontal de banco de dados para melhorar performance e escalabilidade.

**Simulador**: Ferramenta que imita o comportamento de dispositivos reais, permitindo testar aplicações sem hardware físico.

**Sistema**: Conjunto organizado de componentes inter-relacionados que trabalham juntos para atingir objetivos específicos.

**SLA (Service Level Agreement)**: Acordo que define níveis mínimos de qualidade e disponibilidade de serviço.

**SOC 2 (Service Organization Control 2)**: Padrão de auditoria para controles de segurança e disponibilidade de dados.

**Splash Screen**: Tela inicial exibida durante carregamento de aplicativo, geralmente com logo da empresa.

**SQLite**: Sistema de banco de dados relacional leve que funciona como biblioteca incorporada.

**SSL (Secure Sockets Layer)**: Protocolo de segurança para comunicações criptografadas na internet.

**Stakeholder**: Qualquer pessoa, grupo ou organização que pode afetar ou ser afetada pelo desenvolvimento e operação do sistema.

## T

**TalkBack**: Leitor de tela nativo do Android para usuários com deficiências visuais.

**Terraform**: Ferramenta de Infrastructure as Code para provisionamento automatizado de recursos em nuvem.

**Terapia Cognitivo-Comportamental (TCC)**: Abordagem terapêutica que foca na identificação e modificação de padrões de pensamento e comportamento problemáticos.

**Throughput**: Medida de quantidade de trabalho executado por unidade de tempo em um sistema.

**Timestamp**: Marca temporal que registra data e hora exatas de quando evento ocorreu.

**TLS (Transport Layer Security)**: Protocolo criptográfico sucessor do SSL para comunicações seguras na internet.

**Token**: Sequência de caracteres que representa credenciais de autenticação ou autorização.

**Tratamento**: Conjunto de procedimentos e intervenções destinados a ajudar indivíduos a superar problemas de saúde mental ou comportamentais.

## U

**UX (User Experience)**: Experiência do usuário - refere-se à experiência geral que uma pessoa tem ao interagir com um produto ou sistema.

**UI (User Interface)**: Interface do usuário - elementos visuais e interativos através dos quais os usuários interagem com o sistema.

## V

**Vício Comportamental**: Padrão de comportamento compulsivo que interfere significativamente na vida diária de uma pessoa.

**Versionamento**: Processo de atribuir identificadores únicos a diferentes versões de software para controlar mudanças e atualizações.

**VoiceOver**: Leitor de tela nativo do iOS para usuários com deficiências visuais.

## W

**WCAG (Web Content Accessibility Guidelines)**: Diretrizes internacionais para tornar conteúdo web acessível a pessoas com deficiências.

**Widget**: Componente de interface gráfica que permite interação do usuário ou exibe informações no aplicativo.

**Workload**: Quantidade e tipo de trabalho processado por sistema computacional em período determinado.

**Nota**: Este glossário será expandido conforme novos termos técnicos sejam introduzidos durante o desenvolvimento do projeto. Recomenda-se consultar este documento sempre que houver dúvidas sobre terminologia específica utilizada no contexto do sistema betSTOP.
