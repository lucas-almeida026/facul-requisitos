# Glossário

Este glossário define os termos técnicos e conceitos específicos utilizados neste documento de requisitos. Os termos estão organizados em ordem alfabética para facilitar a consulta.

## A

**Android Studio**: Ambiente de desenvolvimento integrado (IDE) oficial para o desenvolvimento de aplicações Android, fornecido pelo Google.

**API (Application Programming Interface)**: Conjunto de definições e protocolos para construir e integrar software de aplicação, permitindo comunicação entre diferentes componentes de software.

**Aplicativo Mobile**: Software desenvolvido especificamente para execução em dispositivos móveis como smartphones e tablets.

## B

**Banco de Dados**: Sistema organizado para coleta, armazenamento e recuperação de informações digitais.

**betSTOP**: Nome do sistema objeto deste documento - aplicativo mobile terapêutico para tratamento do vício em jogos de azar online.

## C

**CI/CD (Continuous Integration/Continuous Deployment)**: Prática de desenvolvimento de software que envolve integração contínua de código e implantação automatizada de aplicações.

**Cloud Computing**: Fornecimento de serviços de computação através da internet, incluindo armazenamento, processamento e software.

**Container**: Tecnologia de virtualização que permite empacotar aplicações e suas dependências em um ambiente isolado e portátil.

**Containerização**: Processo de empacotamento de aplicações em containers para facilitar implantação e escalabilidade.

## D

**Docker**: Plataforma de containerização que permite criar, implantar e executar aplicações em containers.

**Dart**: Linguagem de programação desenvolvida pelo Google, utilizada pelo framework Flutter.

## F

**Flutter**: Framework de desenvolvimento de aplicações móveis criado pelo Google, que permite criar aplicativos nativos para iOS e Android usando uma única base de código.

## G

**GCP (Google Cloud Platform)**: Plataforma de computação em nuvem oferecida pelo Google, fornecendo serviços de infraestrutura, plataforma e software.

**GitHub**: Plataforma de hospedagem de código-fonte que utiliza o sistema de controle de versão Git.

**Git**: Sistema de controle de versão distribuído para rastrear mudanças no código-fonte durante o desenvolvimento de software.

## I

**IDE (Integrated Development Environment)**: Ambiente de desenvolvimento integrado que fornece ferramentas abrangentes para desenvolvimento de software.

**iOS**: Sistema operacional móvel desenvolvido pela Apple para seus dispositivos iPhone e iPad.

## J

**Jogos de Azar**: Atividades que envolvem apostas em eventos com resultados incertos, onde o resultado depende principalmente da sorte.

## M

**MySQL**: Sistema de gerenciamento de banco de dados relacional de código aberto amplamente utilizado.

**Mobile First**: Abordagem de design que prioriza a experiência em dispositivos móveis antes de adaptar para outros dispositivos.

## N

**Notificação Push**: Mensagem enviada por um aplicativo para o dispositivo do usuário, mesmo quando o aplicativo não está ativo.

## P

**Persona**: Representação fictícia de um usuário real, baseada em dados e características de usuários reais, utilizada no design de produtos.

**Psicologia Comportamental**: Área da psicologia que estuda como o comportamento é aprendido e modificado através de estímulos e reforços.

## R

**Redis**: Sistema de armazenamento de estruturas de dados em memória, usado como banco de dados, cache e message broker.

**Reenquadramento Cognitivo**: Técnica terapêutica que envolve identificar e modificar padrões de pensamento negativos ou disfuncionais.

**Requisito Funcional (RF)**: Especificação do que o sistema deve fazer, descrevendo as funções e serviços que o sistema deve fornecer.

**Requisito Não Funcional (RNF)**: Especificação de critérios que podem ser usados para julgar a operação de um sistema, como desempenho, segurança e usabilidade.

## S

**Simulador**: Ferramenta que imita o comportamento de dispositivos reais, permitindo testar aplicações sem hardware físico.

**Sistema**: Conjunto organizado de componentes inter-relacionados que trabalham juntos para atingir objetivos específicos.

**Stakeholder**: Qualquer pessoa, grupo ou organização que pode afetar ou ser afetada pelo desenvolvimento e operação do sistema.

## T

**Terapia Cognitivo-Comportamental (TCC)**: Abordagem terapêutica que foca na identificação e modificação de padrões de pensamento e comportamento problemáticos.

**Tratamento**: Conjunto de procedimentos e intervenções destinados a ajudar indivíduos a superar problemas de saúde mental ou comportamentais.

## U

**UX (User Experience)**: Experiência do usuário - refere-se à experiência geral que uma pessoa tem ao interagir com um produto ou sistema.

**UI (User Interface)**: Interface do usuário - elementos visuais e interativos através dos quais os usuários interagem com o sistema.

## V

**Vício Comportamental**: Padrão de comportamento compulsivo que interfere significativamente na vida diária de uma pessoa.

**Versionamento**: Processo de atribuir identificadores únicos a diferentes versões de software para controlar mudanças e atualizações.

## W

**Widget**: Componente de interface gráfica que permite interação do usuário ou exibe informações no aplicativo.

---

**Nota**: Este glossário será expandido conforme novos termos técnicos sejam introduzidos durante o desenvolvimento do projeto. Recomenda-se consultar este documento sempre que houver dúvidas sobre terminologia específica utilizada no contexto do sistema betSTOP.
