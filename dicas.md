# Sete dicas para a elaboração de um documento de requisitos:

1. concisão: utilize frases curtas.
2. organize os requisitos logicamente (sequência de execução de atividade, por exemplo)
3. organize os requisitos como functionais (RF) and não funcionais (RNF)
4. inicie requisitos obrigatórios com "o sistema dev..." e requisitos desejáveis com "o sistema pode..."
5. verifique os requisitos em busca de possíveis conflitos entre eles.
6. enumer os requisitos e referencie os requisitos do documento de definição de requisitos aos do documento de especificação de requisitos. Deve haver um identificador único para cara requisito (ex. RF001, RNF001)
7. os requisitos não devem apresentar detalhes sobre sua forma de implementação. Eles devem apenas fazer referência ao que o sistema deve fazer e não "como", "quando" ou "por que".
